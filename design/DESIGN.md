# TCM Knowledge Graph Visual Identity

## Style Prompt

Create a dynamic Chinese medicine knowledge system that feels like a living herbarium merged with a graph intelligence interface. The UI should be calm enough for reading dense medical text, but animated enough to feel like an AI/knowledge-graph product. Use rice-paper surfaces, ink-green structure, pale jade panels, cinnabar accents, herb-line ornament, and graph-link pulses. Avoid generic AI SaaS styling.

## Colors

- `#F4EFE3` Rice Paper: main page background.
- `#143D2B` Ink Green: primary navigation, hero depth, graph canvas.
- `#1D2520` Charcoal Ink: primary text.
- `#4E8F5A` Herb Leaf: active controls, graph selected nodes, streaming state.
- `#C8D8B8` Pale Jade: secondary panels and quiet chips.
- `#B9472E` Cinnabar: important CTA, warning, high-signal relation highlight.
- `#E5D7BB` Deco Fiber: borders, separators, herbal paper texture.

## Typography

- Chinese interface text should use a readable system stack: `Microsoft YaHei`, `PingFang SC`, `Source Han Sans SC`, sans-serif.
- Use a restrained serif accent for large English/data labels where available.
- Use tabular numeric styling for graph metrics and model evaluation stats.
- Keep body text at least 15px on web and at least 20px in HyperFrames.

## Layout

- The product is an application, not a landing page. The first viewport should show usable tools.
- Use a left navigation rail on desktop and a bottom/tab navigation on mobile.
- Chat, search results, entity details, and graph controls should be dense but calm.
- Cards may be used for individual repeated entities or messages, with radius no larger than 8px.
- Do not nest cards inside cards.
- Graph visualization should be full-height inside its work area, not a decorative thumbnail.

## Motion

- Ambient motion: slow graph pulses, herb-outline drifting, subtle ink wash reveal.
- Interaction motion: selected graph nodes gently brighten, related edges pulse once, streamed answer text reveals line by line.
- Page transitions: soft wipe inspired by ink spreading across paper.
- Motion must never shift layout or obscure text.

## What Not To Do

- No purple-blue AI gradients.
- No decorative bokeh/orb backgrounds.
- No generic dark slate SaaS dashboard look.
- No oversized marketing hero replacing the actual product.
- No medical diagnosis framing; the product is knowledge exploration and educational QA.

## Screen Requirements

- Assistant desktop: chat, reasoning timeline, graph evidence preview.
- Search + Graph desktop: search field, entity detail panel, relation graph, filters.
- Assistant mobile: compact chat, collapsible reasoning, bottom nav.
- Architecture page: pipeline diagram and model replacement story.
